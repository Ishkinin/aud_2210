Лабораторная-работа-3¶

Цель работы¶

  1. Изучение системных вызовов для обмена данными между процессами.
  2. Приобретение навыков написания и трансляции системного ПО на языке C.
  3. Изучение утилиты make и ее использование для трансляции программ на языке C.
  4. Написать программу для обмена текстовыми сообщениями между процессами, с использованием механизма разделяемой памяти.
  5. Обеспечить синхронизацию обмена с помощью механизма семафоров.

Выполнение работы¶

Для компиляции написанной программы был использован контейнер. Контейнер строится с использованием Dockerfile.:

Программы в процессе построения образа контейнера помещаются в файловую систему образа, производится ее сборка и установка (исходные файлы._).

Для демонстрации работы программы контейнер используется команда:

docker run -it --rm ravil/abc_linux_lab6

Следом после этого запускается скрипт._:

Скрипт запускает программу-сервер lab6-server и несколько раз отправляет сообщения серверу с помощью утилиты lab6-client. Сервер останавливает свое выполнение после получения пяти сообщений. После остановки сервера можно видеть уведомление от lab6-client о том, что сервер не запущен.

Выводы¶

  1. благодаря технологии контейнеров имеется возможность запуска процессов в изолированном окружении абсолютно под разными операционными системами.
  2. Для того, чтобы реализовать механизм разделяемой памяти, стоить применять например высосокоуровневую библиотеку 0mq для Go.
