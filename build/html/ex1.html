Лабораторная-работа-1¶

Цель работы¶

  1. Приобретение навыков по управлению процессами.
  2. Изучение системного вызова fork().
  3. Приобретение навыков написания и трансляции системного ПО на языке C.
  4. Изучение утилиты make и ее использование для трансляции программ на языке C.
  5. Использование удаленных серверов для трансляции приложений.

Выполнение работы¶

Запуск DOCKER-контейнера из образа ubuntu 14.04 в интерактивном режиме

sudo docker run -it --rm --name lab4 ubuntu:14.04

В контейнере создаем каталог work и переходим в него.

mkdir work
cd /work

Для выполнения работы неоднобимо установить недостающие компаненты:

Компилятор gcc

apt-get install gcc

Редактор nano

apt-get install nano

Утилита make

apt-get install make

SSH клиент

apt-get install -y openshh-clients

В лабораторной работе используются файлы, хранящиеся на сервере лаборатории. Скопирум данные файлы в контенейр выполнив команду:

scp -r -P 6666 student@openit.guap.ru:/container/ABC-Linux/lab4/* work/

Просмотреть исправленные файлы возможно, перейдя по ссылкe.

Запуск отранслированной программы

./work/bin/lab4

Теперь сохраняем наш контейнер и заливаем и его на dockerhub.

Команды:

docker commit lab4
docker push lab4 ravil/abc-linux-lab4

Выводы¶

  1. Приобрел навыки управления процессами.
  2. Благодаря технологии контейнеров имеется возможность запуска процессов визолированном окружении абсолютно под разными операционными системами.
  3. Приобрел навыки управления процессами и навыки написания и перевода системного программного обеспечения на языке C.
  4. Изучил утилиту make и ее использование для перевода программ.
  5. И обнаружил что не всегда getppid() возвращает номер породившего процесса.
